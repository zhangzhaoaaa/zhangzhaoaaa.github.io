webpack的主要配置参数分析

发表于 2017-07-07   |   分类于 webpack

我们知道webpack的核心概念就是

  1. Entry
  2. Output
  3. Loader
  4. Plugin

所以我们依然主要围绕着这几个核心概念进行主要参数的分析，plugin属性本身没什么配置，就是一个插件数组，所以我们先按下不表，之后有plugin插件专题会讨论

Entry

这是应用程序的入口
根据不同的业务场景，会有单页应用和多页应用，那么该如何处理呢

1.单页应用

用法： entry: string | Array
例子：

2.多页应用

用法： entry: {[entryChunkName: string]: string | Array}

Output

我们看到多页应用的时候，输出文件是[name]-[hash].js，这就是为了处理多入口文件的一种方式。
我们就来说明一下filename，这些属性是什么意思。

1.filename

模板 说明
[hash] module的标识的hash
[chunckhash] chunk内容的hash
[name] module名称
[id] module标识
[file] module文件名
[filebase] 模块的basename
[query] 模块query，文件名?后面的字符串

2.path
输出bundle路径

3.hashDigest
生成hash值的编码，默认使用hex。
4.hashDigestLength
hash数字的长度，默认20.
5.library
这个一般在写JavaScript库的时候使用，这个是library的名字
6.libraryTarget
配置这个library如何暴露出来。
默认export出来是var，当你的library加载的时候，返回的你entry的值要赋值给一个变量

当然还 有”this”,”window”,”global”,”commonjs”,”commonjs2”,”amd”,”umd”,”assign”,”jsonp”
我们一般使用umd就可以了，包含了commonjs,amd和全局变量方式。

7.publicPath
当使用按需加载或者加载外部资源的时候，如图片，文件等，这个选项很重要啊。当设置了不正确的值，你加载这些资源的时候，会收到404的错误。
当output目录在浏览器中引用的时候，这个参数指定了输出目录的public URL.
这个默认值是“”

例子：

一个chunk的请求如同/assets/4.chunk.js

我们之后在devServer的时候会再次说明这个属性

Loaders（Module）

loaders是转换成webpack认识的模块的配置

1.module.noParse
用法： RegExp | [RegExp] | function
会阻止webpack解析匹配上的文件。忽略import,require,define或者其他引入模块机制。忽略这些大的包会提升编译性能。
2.moduel.rules
是一个规则数组

Rule(规则)

一个规则会被分成三个部分——Conditions（条件），Results(结果)和nested Results（内置结果）

Rule Conditions

对于条件有两个输入：

  1. 资源文件
  2. 资源所被应用的地方

在app.js中我们 import “./style.css”,资源是/path/to/style.css,提供者是/path/to/app.js

在Rule里的属性，test，include，exclude和resource对应了资源

Rule results

当Rule条件匹配了，rule的results就可以被用了。
一个Rule会有两个参数：

  1. loaders: 一个loaders的数组来应用到resource上
  2. parser参数： 这些parser在生成module时所需要的参数

参数有loader, options, use，兼容参数有query,loaders

Nested Rules

Rule.enforce: “pre”|”post”
指定了loader的分类。没有值意味着是normal loader

所有loaders的排序是post inline normal pre

所有normal loaders可以使用前缀!触发
所有normal和pre loaders可以用-！触发
所有normal ,post,pre loaders可以用!!触发
Inline loaders和!前缀不应该被用，因为它们是非标准的。

Rule

1.Rule.use
UseEntries的一个列表。
UseEntry是个对象，必须有一个loader属性是字符串，有一个options属性是字符串或者对象。这个值会被传递给loader，在解析的时候使用。

Condition

  1. 一个字符串： 匹配一个提供的字符串。比如：一个绝对路径或者一个文件的绝对路径
  2. 正则表达式：匹配输入
  3. 一个方法：必须返回一个真值来匹配
  4. 一个条件数组：至少有一个条件匹配
  5. 一个对象：所有属性匹配。

{test: condition}:提供一个正则或者正则数组，
include：condition 字符串、字符串数组
exclude:condition 不要匹配，字符串、字符串数组
and :[condition] 所有条件匹配
or:[condition] 所有条件匹配
not:[condition] 所有条件不匹配

resolve

  1. extensions 处理的文件扩展（后缀）
  2. alias 给资源起个别名

1. what is webpack

webpack是一个模块打包工具，将依赖的模块打包成最后的包含这些模块的静态资源。
当webpack处理你的项目的时候，它会递归构建一个你项目里的每个模块的依赖图，然后打包这些模块到少量的会被浏览器加载的静态资源中（通常是一个）

2. what is its goal

它出来的原因是已有的工具不能很好的满足大型应用的开发（尤其是大的单页应用）
它当时出来是为了达成以下几个目的：

  1. 将依赖树拆分到chunked（块）里按需加载
  2. 保证初始化时间短
  3. 每个静态资源都是个模块
  4. 能够集成第三方库作为模块
  5. 尽可能的定制化模块加载器的每一个部分
  6. 满足大型项目

3. what is module

既然webpack是将每个资源当成了module处理，那么我们首先得知道什么是模块化编程：
模块化编程是一种软件设计技术，强调的是把一个程序的功能拆分到独立的，可交互的模块中，就比如说每一个模块包含的是整个应用程序的一个方面

每一个模块比起整个项目来说，有更小的层面来做验证，调试和测试。写的好的模块能够提供可靠的抽象和封装边界，以至于说在整个项目中，每一个模块有一致性的设计和清晰的目的

Nodejs在最开始的时候就支持了模块化编程。但是在web端，模块化支持的相对慢一点。在web端有多个不同收益和限制的工具支持模块化。webpack其实是学习了这些工具，然后将这些模块化的概念应用到了项目中来。

webpack支持的模块：

  1. ES2015 中的import
  2. CommnonJS require()
  3. AMD define 和 require
  4. @import 在css/sass/less
  5. image的url，url(…)或者<img src=...>文件

支持的不同语言和预处理器的模块类型也很多

  1. CoffeeScript
  2. TypeScript
  3. ESNext(Babel)
  4. Sass
  5. Less
  6. Stylus

等等，可以自定义

4. 核心概念

我们需要了解webpack如何使用的话，首先得了解其核心概念

  1. entry
  2. output
  3. loaders
  4. plugins
Entry

webpack创建了一个应用的依赖图。这个图的起点就是entry，entry告诉webpack从哪开始，并且按这个依赖图就知道打包什么模块了。你可以把这个entry作为你的项目的根节点或者是启动你应用的第一个文件。

Loaders

它的目的是让你项目里的所有的资源变成webpack关心的而不是浏览器关心的东西。webpack认为每一个文件都是一个模块，css,html,scss,jpg等等，但是webpack仅仅理解javascript。

在webpack中Loaders把这些文件转换成modules，它们会被加到你的依赖图中。

在webpack配置中，它们有两个目的

  1. 指定什么文件应该被这个特定的loader处理(test属性)
  2. 转换文件能够加入到依赖图中（最终到bundle中）(use属性)

Plugins

我们看到Loaders是在每一个文件的基础上执行转换，plugins大多被用在你的bundled模块在“compilations”和”chunks”的时候，执行一些操作和做一些定制化的功能。webpack的插件系统是非常强大和可定制化的。

使用一个插件，你需要require它，把它加到plugins的数组中。大多数插件是可以使用参数配置的。在一个配置里，为了不同的目的，你可以使用一个插件多次，你需要使用new来创建这个插件的实例

Event loops

发表于 2017-03-16   |   分类于 JS基础

为了协调events,用户交互，scripts,渲染，网络请求等等，用户代理必须使用event loops
有两种event loops:浏览器上下文和workers
每个用户代理必须至少有一个浏览器上下文的event loop
event loop有一个或者多个task queues.一个task queue是一个有序的tasks列表，这个算法负责以下工作：

Events

分发一个Event对象到一个特别的EventTarget对象会被一个特别的task来处理
注意：不是所有的事件被task queue来分发，在其他tasks中，很多也被分发

DOM的操作处理

当一个用户代理要将一个task加入队列，那么必须指明在相关的event loop中的指定的task queues中。

每一个任务被定义为来自一个特别的task source。所有来自一种task source的tasks目标是一种类型的event loop,它们总是被增加到同样的task queue里，但是来自不同的task sources的tasks会放置到不同的task queues里。

每一个event loop有个当前运行的task。初始化时，这个是null。每一个event loop有个performing a microtask checkpoint 标志，这个标志初始化为false。被用来阻止performing a microtask checkpoint算法的重复调用。
